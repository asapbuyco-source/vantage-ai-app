## 1. Executive Summary & Audit Findings

### 1.1 Architecture Overview

Vantage AI is a sports prediction SaaS with:
- **Frontend**: React 19 + Vite + Tailwind CSS + Framer Motion
- **Backend**: Node.js/Express (`server.js`) + Python Quant Engine (`backend/quant/`)
- **Data**: Firebase/Firestore (primary), Supabase (secondary)
- **APIs**: Sportmonks (data), Gemini (fallback AI), OpenAI (grading fallback)
- **Scheduler**: `node-cron` in `backend/scheduler.js`

### Phase 1: Quant Engine Math Fixes (Python)

#### Task 1.1 — Fix Missing Probability Fields

**Target File:** `backend/quant/probability_engine.py`

**Problem:** `CombinedProbabilities` is missing `over15`, `under15`, `over35`, `under35`, `btts_and_over25`, `draw_no_bet_home`, `draw_no_bet_away`. The `ev_engine.py` references them via `MARKET_TO_PROB` but gets `None`.

**Action:** 
1. Add to the `CombinedProbabilities` dataclass after existing fields:
```python
over15: float = 0.0
under15: float = 0.0
over35: float = 0.0
under35: float = 0.0
btts_and_over25: float = 0.0
draw_no_bet_home: float = 0.0
draw_no_bet_away: float = 0.0
```

2. In `compute_combined()`, after computing `btts`/`btts_no`, add:
```python
over15 = poisson_probs.get("over15", 1.0 - poisson_probs.get("under15", 0.15))
under15 = 1.0 - over15
over35 = poisson_probs.get("over35", 0.20)
under35 = 1.0 - over35
btts_and_over25 = btts * over25
if home_win + away_win > 0:
    draw_no_bet_home = home_win / (home_win + away_win)
    draw_no_bet_away = away_win / (home_win + away_win)
else:
    draw_no_bet_home = 0.5
    draw_no_bet_away = 0.5
```

3. Include these in the returned `CombinedProbabilities(...)` call.

**Validation:** Run `python probability_engine.py`. All fields print without `None`. Verify `over15 + under15 ≈ 1.0`.

#### Task 1.2 — Add Missing Poisson Probabilities

**Target File:** `backend/quant/poisson_model.py`

**Problem:** `compute_poisson_probs()` returns `over25`/`under25` but not `over15`/`under15`/`over35`/`under35`.

**Action:** After computing `over25`/`under25` from the score grid, add:
```python

over35 = 1.0 - under35
```
Include all four in the returned dict.

**Validation:** Run with mu_home=1.5, mu_away=1.2. Verify `over15 > 0.7`.

#### Task 1.3 — Fix Double Kelly Fraction

**Target File:** `backend/quant/quant_pipeline.py`

**Problem:** Lines 296-300 apply `kelly_fraction = 0.25` AND `kelly_stake_pct()` internally uses `KELLY_FRACTION = 0.25`. Net result: 1/16 Kelly.

**Action:** Change lines 296-300 from:
```python
kelly_fraction = 0.25
base_kelly = kelly_stake_pct(best_bet.model_prob, best_bet.odds)
kelly = round(max(0, base_kelly * staleness_mult * kelly_fraction), 2)
```
TO:
```python
base_kelly = kelly_stake_pct(best_bet.model_prob, best_bet.odds)
kelly = round(max(0, base_kelly * staleness_mult), 2)
```

**Validation:** For prob=0.63, odds=1.85, kelly should be ~3.5%, not ~0.88%.

#### Task 1.4 — Fix Division-by-Zero in Form Model

**Target File:** `backend/quant/form_model.py`

**Problem:** `_performance_score()` divides by `(xg + poss + sot)`. All-zero inputs return 0.0 instead of neutral 0.5.

**Action:** Add guard at top of function:
```python
def _performance_score(xg: float, poss: float, sot: float) -> float:
    if xg == 0 and poss == 0 and sot == 0:
        return 0.5
    # ... rest of existing logic
```

**Validation:** `_performance_score(0, 0, 0)` returns `0.5`.

#### Task 1.5 — Fix Elo K-Factor Floor

**Target File:** `backend/quant/elo_rating.py`

**Action:** In `get_k_factor()`, ensure minimum K of 20:
```python
def get_k_factor(league_tier: int, match_importance: float = 1.0) -> float:
    base_k = {1: 32, 2: 28, 3: 24, 4: 20}.get(league_tier, 20)
    return max(20, base_k * match_importance)
```

**Validation:** Tier 4 league returns K=20, not 15.

### Phase 5: Accumulator Correlation Fix

**Target File:** `backend/quant/accumulator_engine.py`

```python
league_counts = {}
for l in self.legs:

```
